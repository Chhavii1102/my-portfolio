College Management System

Description

The College Management System(CMS) is a Python and MySQL-based project that helps in managing college data digitally in a secure and organized way.It provides different access levels for Admin,Teacher, and Student, making the system safe and reliable.

The main goal of this project is to reduce manual work, maintain accuracy, and make information easily accessible to the right users.

User Roles & Features

  1. Admin (Full Control)

    2. The admin has complete control over the system:

    • Can add, update, and delete teacher and student records.
    • Approves or rejects signup requests from teachers and students.
    • Views all reports and system activity.

  2. Teacher (Student Data Control)

    • Can view and manage full details of all students.
    • Update or modify student records.
    • Cannot access admin-level settings.

  3. Student (Secure Limited Access)

    • Students sign up and then log in with their own account.
    • Can only view their personal data.
    • Cannot see other students' data, making the system secure and private.

Extra Key Features

Role-Based Login & Signup

- Secure login for Admin, Teacher, and Student with separate dashboards.

Requests Management

- Signup requests go to admin for approval to keep the system safe.

Data Export

- Export data to a text file for reports and backups.

Search

- Quickly find any student or teacher by their roll no or employee id.

Secure Access

- Students cannot edit or view others' data.
- Teachers have limited permissions compared to Admin.

Technology Used

  • Frontend(GUI): Python Tkinter
  • Backend(Database): MySQL
  • Language: Python
  • File Handling : For export data
